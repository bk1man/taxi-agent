## 零、司机端设计（重大更新）

### 0.1 核心设计理念

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   传统司机：                                                     │
│   打开App → 盯屏幕等单 → 看到订单 → 手动抢 → 手动沟通            │
│   问题：累、慢、容易错过订单                                      │
│                                                                 │
│   司机 Agent：                                                  │
│   设置偏好 → 喝咖啡 → Agent自动抢单 → 有事才叫你                  │
│   优点：轻松、快速、一个顶三个                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 0.3 自动接单规则引擎

```python
class DriverPreference:
    """司机偏好设置"""
    
    # 基础设置
    max_distance_km: float = 5.0          # 最远接单距离
    min_fare_yuan: float = 20.0          # 最低接单价
    auto_accept: bool = True              # 是否自动接单
    
    # 区域设置
    allowed_areas: List[str] = []        # 允许接单区域
    blocked_areas: List[str] = []         # 禁止接单区域
    
    # 时间设置
    work_hours: List[Tuple[int, int]] = [(8, 20)]  # 工作时段
    auto_offline_outside_hours: bool = True  # 工作时间外自动下线
    
    # 偏好设置
    car_type_preference: str = "舒适"     # 偏好车型
    long_trip_ratio: float = 0.3         # 长途订单比例(0-1)
    rush_hour_multiplier: float = 1.5     # 高峰期加价接单阈值


class OrderEvaluator:
    """订单评估器"""
    
    def evaluate(self, order: Order, driver: Driver) -> EvaluationResult:
        """
        评估订单是否值得接
        返回: (得分, 是否接单, 原因)
        """
        reasons = []
        score = 100
        
        # 1. 距离检查
        if order.distance_km > driver.preference.max_distance_km:
            return EvaluationResult(
                score=0, 
                accept=False, 
                reason=f"距离{order.distance_km}km超过上限{driver.preference.max_distance_km}km"
            )
        score -= order.distance_km * 5  # 每公里扣5分
        reasons.append(f"距离{order.distance_km}km")
        
        # 2. 价格检查
        if order.estimated_fare < driver.preference.min_fare_yuan:
            return EvaluationResult(
                score=0,
                accept=False,
                reason=f"价格{order.estimated_fare}元低于最低要求"
            )
        profit_ratio = (order.estimated_fare - order.distance_km * 2) / order.estimated_fare
        score += profit_ratio * 30  # 利润空间加分
        reasons.append(f"利润率{profit_ratio:.0%}")
        
        # 3. 区域检查
        if driver.preference.allowed_areas:
            if not any(area in order.destination_address for area in driver.preference.allowed_areas):
                return EvaluationResult(
                    score=0,
                    accept=False,
                    reason="目的地不在允许区域内"
                )
        
        # 4. 乘客评分检查
        if order.passenger_rating < 4.5:
            score -= (4.5 - order.passenger_rating) * 20
            reasons.append(f"乘客评分{order.passenger_rating}")
        
        # 5. 长途订单加分
        if order.distance_km > 20:
            score += 20
            reasons.append("长途订单+20分")
        
        # 6. 高峰期加成
        if self._is_rush_hour():
            if order.estimated_fare >= driver.preference.min_fare_yuan * driver.preference.rush_hour_multiplier:
                score += 15
                reasons.append("高峰期加价")
        
        # 最终决策
        accept = score >= 60  # 得分60以上自动接
        
        return EvaluationResult(
            score=score,
            accept=accept,
            reason=" | ".join(reasons)
        )
```

## 二、数据模型详解

### 2.1 司机表 (drivers)
```sql
CREATE TABLE drivers (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '司机ID',
    name            VARCHAR(50) NOT NULL COMMENT '姓名',
    phone           VARCHAR(20) UNIQUE NOT NULL COMMENT '手机号',
    avatar_url      VARCHAR(500) COMMENT '头像URL',
    
    -- 车辆信息
    car_model       VARCHAR(100) COMMENT '车型，如"宝马5系"',
    car_number      VARCHAR(20) UNIQUE NOT NULL COMMENT '车牌号，如"京A12345"',
    car_color       VARCHAR(20) COMMENT '车辆颜色',
    car_seats       INT DEFAULT 4 COMMENT '座位数',
    
    -- 位置信息（实时更新）
    latitude        DECIMAL(10, 8) COMMENT '当前纬度',
    longitude       DECIMAL(11, 8) COMMENT '当前经度',
    location_time   DATETIME COMMENT '位置更新时间',
    
    -- 状态
    status          ENUM('online', 'busy', 'offline') DEFAULT 'offline' COMMENT '在线状态',
    
    -- 评分
    rating          DECIMAL(2, 1) DEFAULT 5.0 COMMENT '评分',
    total_orders    INT DEFAULT 0 COMMENT '完成订单数',
    
    -- 元数据
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_location (latitude, longitude),
    INDEX idx_status (status)
) COMMENT '司机表';
```

### 2.2 乘客表 (passengers)
```sql
CREATE TABLE passengers (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '乘客ID',
    openid          VARCHAR(100) UNIQUE NOT NULL COMMENT '微信OpenID',
    nickname        VARCHAR(100) COMMENT '微信昵称',
    avatar_url      VARCHAR(500) COMMENT '头像URL',
    phone           VARCHAR(20) COMMENT '手机号',
    
    -- 默认地址
    home_address    VARCHAR(200) COMMENT '家庭地址',
    home_lat        DECIMAL(10, 8) COMMENT '家庭地址纬度',
    home_lng        DECIMAL(11, 8) COMMENT '家庭地址经度',
    company_address VARCHAR(200) COMMENT '公司地址',
    company_lat      DECIMAL(10, 8) COMMENT '公司地址纬度',
    company_lng      DECIMAL(11, 8) COMMENT '公司地址经度',
    
    -- 统计数据
    total_orders    INT DEFAULT 0 COMMENT '历史订单数',
    
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_openid (openid)
) COMMENT '乘客表';
```

### 2.3 订单表 (orders)
```sql
CREATE TABLE orders (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '订单ID',
    order_no            VARCHAR(32) UNIQUE NOT NULL COMMENT '订单编号',
    
    -- 乘客信息
    passenger_id        BIGINT NOT NULL,
    pickup_address      VARCHAR(200) NOT NULL COMMENT '出发地址',
    pickup_lat          DECIMAL(10, 8) NOT NULL COMMENT '出发地纬度',
    pickup_lng          DECIMAL(11, 8) NOT NULL COMMENT '出发地经度',
    
    -- 目的地信息
    destination_address VARCHAR(200) NOT NULL COMMENT '目的地地址',
    destination_lat     DECIMAL(10, 8) NOT NULL COMMENT '目的地纬度',
    destination_lng     DECIMAL(11, 8) NOT NULL COMMENT '目的地经度',
    
    -- 距离与费用
    distance            DECIMAL(8, 2) COMMENT '行程距离(km)',
    estimated_fare     DECIMAL(8, 2) COMMENT '预估费用(元)',
    actual_fare         DECIMAL(8, 2) COMMENT '实际费用(元)',
    
    -- 司机信息
    driver_id           BIGINT COMMENT '接单司机ID',
    driver_lat          DECIMAL(10, 8) COMMENT '派单时司机位置纬度',
    driver_lng          DECIMAL(11, 8) COMMENT '派单时司机位置经度',
    
    -- 派单信息
    dispatch_distance   DECIMAL(6, 2) COMMENT '派单距离(km)',
    dispatch_time       DATETIME COMMENT '派单时间',
    
    -- 状态流转
    -- pending → matched → picked → completed/cancelled
    status              ENUM('pending', 'matched', 'picked', 'completed', 'cancelled') DEFAULT 'pending',
    status_message      VARCHAR(200) COMMENT '状态说明，如取消原因',
    
    -- 时间戳
    created_at          DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '订单创建时间',
    matched_at          DATETIME COMMENT '司机接单时间',
    picked_at           DATETIME COMMENT '乘客上车时间',
    completed_at        DATETIME COMMENT '订单完成时间',
    cancelled_at         DATETIME COMMENT '订单取消时间',
    
    -- 支付
    payment_status       ENUM('unpaid', 'paid', 'refunded') DEFAULT 'unpaid',
    payment_time        DATETIME COMMENT '支付时间',
    transaction_id      VARCHAR(64) COMMENT '微信支付单号',
    
    -- 乘客备注
    passenger_note      VARCHAR(200) COMMENT '乘客备注',
    
    FOREIGN KEY (passenger_id) REFERENCES passengers(id),
    FOREIGN KEY (driver_id) REFERENCES drivers(id),
    
    INDEX idx_passenger (passenger_id),
    INDEX idx_driver (driver_id),
    INDEX idx_status (status),
    INDEX idx_created (created_at)
) COMMENT '订单表';
```

### 2.4 司机位置历史表 (driver_location_history)

```sql
CREATE TABLE driver_location_history (

    recorded_at     DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_driver_time (driver_id, recorded_at)
) COMMENT '司机位置历史，用于轨迹追踪';
```

## 三、Agent 工具详解

### 3.1 find_nearby_drivers

**工具描述**：查询指定位置附近的可用司机

**输入参数**：
```json
{
  "latitude": 39.908823,      // 必填，乘客纬度
  "longitude": 116.397470,      // 必填，乘客经度
  "radius_km": 3,              // 可选，搜索半径，默认3公里
  "limit": 5,                  // 可选，返回数量，默认5
  "car_type": "舒适"           // 可选，车型偏好
}
```

**返回结果**：
```json
{
  "success": true,
  "count": 3,
  "drivers": [
    {
      "driver_id": 1001,
      "name": "张师傅",
      "phone": "138****1234",
      "avatar_url": "https://...",
      "car_model": "宝马5系",
      "car_number": "京A12345",
      "car_color": "黑色",
      "rating": 4.9,
      "distance_km": 0.8,
      "eta_minutes": 3,
      "total_orders": 1256
    },
    // ...更多司机
  ]
}
```

**算法逻辑**：
1. 查询 status='online' 的司机
2. 计算司机与乘客的距离（ Haversine 公式）
3. 按距离排序，返回最近的 N 辆
4. 计算 ETA = distance / 平均车速

### 8.2 派单算法

```python
def dispatch_order(order_id, preferred_driver_id=None):
    """
    派单算法
    优先级：
    1. 如果指定了司机且司机空闲 → 派给该司机
    2. 否则 → 派给距离最近的空闲司机
    """
    order = get_order(order_id)
    passenger_location = (order.pickup_lat, order.pickup_lng)
    
    if preferred_driver_id:
        driver = get_driver(preferred_driver_id)
        if driver.status == 'online':
            return assign_driver(order, driver)
    
    # 查询附近空闲司机
    nearby_drivers = find_nearby_drivers(
        lat=passenger_location[0],
        lng=passenger_location[1],
        radius_km=5,
        status='online'
    )
    
    if not nearby_drivers:
        return None  # 无可用司机
    
    # 按距离排序，选择最近的
    selected = sorted(nearby_drivers, key=lambda d: d.distance)[0]
    return assign_driver(order, selected)
```

### 8.3 费用计算

```python
def calculate_fare(distance_km, car_type='舒适'):
    """
    费用计算规则：
    - 起步价：10元（包含3公里）
    - 里程费：2元/公里（超出3公里部分）
    - 车型系数：舒适1.2倍
    """
    base_price = 10
    per_km_price = 2
    type_multiplier = {'经济': 1.0, '舒适': 1.2, '豪华': 1.5}.get(car_type, 1.0)
    
    if distance_km <= 3:
        fare = base_price
    else:
        fare = base_price + (distance_km - 3) * per_km_price
    
    fare = fare * type_multiplier
    return round(fare, 2)
```

```
  - 车辆性质变为"预约出租客运"
  - 商业保险（承运人责任险）
```

### 18.2 用户协议要点

```
《乘客协议》必须包含：
1. 服务说明与费用透明
2. 取消与退款政策
3. 隐私保护说明
4. 免责声明（违法行为/违禁品等）
5. 争议解决机制

《司机协议》必须包含：
1. 服务标准与违规扣罚
2. 费用结算与提现规则
3. 保证金说明
4. 取消接单限制
5. 投诉处理流程
```
